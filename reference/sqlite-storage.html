SQLite Storage Reference

Table Purpose Important Safety Detail
employees Employee identity and active status. No images, only employee metadata.
face_embeddings Numeric face vectors tied to employees. Stores vector_json, not photos or frame paths.
attendance_events Clock-in and clock-out events. Stores employee ID, timestamp, event type, confidence, and match distance.

Repository Methods

  • initialize_database(path): create schema.
  • add_employee(employee): persist an employee record.
  • add_embedding(employee_id, embedding): persist a numeric embedding.
  • add_attendance_event(event): persist an attendance log entry.
  • list_table_columns(): inspect schema for safety tests.

Rules

  • Enable SQLite foreign keys on every connection.
  • Wrap database failures in StorageError.
  • Keep raw camera frames in memory only.
