SQLite Storage Boundaries

Phase 3 teaches how validated data becomes durable data without leaking raw biometric images.

The Small Win

We add a SQLite repository that stores employees, face embeddings, and attendance events. Future modules call repository methods instead of writing SQL directly.

Why a Repository?

The repository keeps persistence decisions in one place: table names, foreign keys, JSON encoding, and database errors. Recognition code should think in terms of employees, embeddings, and events, not SQL strings.

Memory hook: storage is a border checkpoint. It persists only what the contracts allow and the schema names.

Security Shape

The schema stores numeric embedding vectors as JSON text. It does not include columns for images, photos, raw bytes, or frame paths. That matters because schema design makes unsafe behavior harder to accidentally introduce later.
