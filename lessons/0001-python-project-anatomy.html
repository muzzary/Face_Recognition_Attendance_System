Python Project Anatomy

Phase 1 teaches how a Python repo becomes an installable project without adding app dependencies yet.

The Small Win

After this phase, the package can be installed in editable mode with python -m pip install -e .. That means tests import face_attendance the same way the application will import it later.

Three Files Matter

pyproject.toml is the project control panel. The Python Packaging User Guide describes it as the place for build-system settings, project metadata, dependencies, and tool configuration.

src/face_attendance/ is the import package. The src layout keeps tests from accidentally importing code just because the repository root is on Python's path.

tests/ is the feedback loop. For now we keep unittest because it is in the Python standard library and avoids a new dependency.

Memory hook: metadata says what the project is, source says what it does, tests prove what still works.

What We Avoided

We did not add OpenCV, Pydantic, or a face-recognition library in this phase. Those are real dependencies and should be chosen deliberately because they affect Windows setup, CI speed, and the final demo.
