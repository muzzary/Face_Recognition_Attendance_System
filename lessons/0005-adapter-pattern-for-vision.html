Lesson 5: The Adapter Pattern for Vision Libraries

Objective: see how an adapter isolates a third-party detector so the rest of the app stays testable and swappable.

Key ideas

  • FaceDetector is a protocol (structural interface): anything with detect(frame) -> list[DetectedFace] qualifies — the real YuNet adapter or a test fake.
  • The adapter converts raw library output (a float matrix) into validated contracts at the boundary: clamping boxes to the frame, dropping degenerate boxes, clamping scores. Garbage stops here.
  • Model files are data, not code: they are downloaded by a pinned, hash-verified script and gitignored — never committed.
  • Swapping YuNet for another detector later means writing one new adapter; matching, liveness, and storage never change.

Retrieval question

Why does the detector return an empty list for zero faces but raise DetectionError for a corrupt frame? (Answer: zero faces is a normal outcome the pipeline must handle; a corrupt frame means the input contract was violated and continuing would hide a real bug.)
