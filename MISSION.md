# Mission: Face Recognition Attendance System

## Why

Build and understand a secure biometric attendance system well enough to defend the architecture, implementation tradeoffs, and limitations in an internship review conversation.

## Success looks like

- Ship a working Python attendance pipeline from camera capture to secure clock-in/clock-out logging.
- Explain why each module exists and how data moves through the system.
- Demonstrate enrollment, recognition, liveness rejection, and smooth background processing.
- Discuss biometric data safety honestly, including what the system stores and what it never stores.

## Constraints

- Work phase by phase, with tests, self-review, commits, and user manual testing checkpoints.
- Keep dependencies minimal and deliberate.
- Use the project brief as the source of truth for deliverables.
- Teach one small concept per phase so implementation and learning move together.

## Out of scope

- Production-grade enterprise access control.
- Cloud deployment before the local system works end to end.
- Claims that liveness detection is perfect or impossible to bypass.
